# ***语言编码规范&约束模板

# 1 条款说明

规范中的**四级标题**（`####`）代表一个具体条款，命名规则如下：

- **规则（Guideline）**：编号格式为 `G.Element.NN`，其中 `Element` 为对应二级目录领域的3位英文缩略语，`NN` 为从01开始的2位数字编号。例如 `G.NAM.01`。
- **原则（Principle）**：编号格式为 `P.NN`，例如 `P.01`。

每个条款均有对应的**级别**：
- **要求**：产品原则上应遵从。
- **建议**：属于最佳实践，产品可结合业务情况决定是否纳入。

> 本文件中仅保留了各条款的**编号与标题**。如需查阅某条款的完整描述、原理说明、正例与反例，请按需查阅对应的参考文件，路径规则如下：
> - 规则条款：`references/G.XXX.NN.md`（例如 `references/G.NAM.01.md`）
> - 原则条款：`references/P.NN.md`（例如 `references/P.01.md`）

# 2 代码风格

代码风格一般包含标识符的命名风格、排版与格式风格，注释风格。一致的编码习惯与风格，会使代码更易阅读、更易理解、更易维护。统一的代码风格也是一致性原则最直观的体现。
