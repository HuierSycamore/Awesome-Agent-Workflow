# 软件变更影响分析自检 Checklist

## 1. 使用说明

本 Checklist 与软件变更影响分析配套使用，供设计和开发人员在设计提交、代码合入及发布前自检。

结果栏填写：

- 通过：已经分析或实现，且证据满足要求；
- 不通过：存在缺失或不符合项；
- N/A：不适用，必须在证据或问题栏说明原因；
- 待确认：证据不足，必须指定责任人和完成时间。

证据位置可填写设计章节、接口名、代码文件和行号、提交或 PR、调用链搜索结果、测试用例、日志、性能原始数据或评审结论。

门禁规则：

- 任一检查项不得只填"通过"而不提供证据。
- 存在 E3 非兼容变更但未获得批准、未提供迁移或回退措施时，不得通过。
- 存在未关闭的 P0、P1 问题时，原则上不得合入或发布。
- 对外行为、上层影响、内存或性能标记"待确认"时，不得形成最终 E0 结论。
- "接口签名未变""单元测试通过"或"代码改动很小"均不能单独证明无对外影响。
